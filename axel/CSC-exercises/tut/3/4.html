PROOF (2/3)

That worked, and it was faster - but it wasn't impressively fast. What about running the same selector on a whole cluster? We have set up a PROOF "virtual machine" cluster for you: it uses one core of all the exercise machines. Depending on when you get here the PROOF cluster might be pretty busy or completely empty, and you will get a performance improvement that depends on the available resources.

So let's run it! The PROOF master is called "server". Any speed difference?

The lecture tells you how to initialize a PROOF session. You will need to give the master as an argument. Don't forget to tell the TChain to use PROOF for Process(). And yes, that's of course also in the lecture.

If you have a problem connecting to the PROOF server please try again. If the problem persists we might have to restart the server... so let us know!

To start PROOF run TProof::Open("server"). It will tell you the number of nodes that it connected for you. Then inform the TChain that it should use PROOF, e.g. by adding the line ch->SetProof() to your chain setup script. Now you run the selector like before: ch->Process(...) and it will use PROOF!
