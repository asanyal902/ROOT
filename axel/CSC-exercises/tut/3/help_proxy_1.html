3 My Second TSelector [OPTIONAL!]

New create your own TSelector - this time we use a different mechanism called Proxy. You can create a skeleton proxy-based TSelector from a TTree by calling TTree::MakeProxy(...). Modify it so it does what the previous selector did (you may remove the call to Sleep() now ;-). This kind of selector is powerful because it determines itself what part of the TTree must be read, without the need to specify branches etc.

Check the documentation for TTree::MakeProxy().
