2 Reading a Tree

Now you use TTree::Draw() again, but on a non-split tree (split-level 0). The tree can be found in the file TFile::Open("/var/www/html/ROOT/exercise2_nonsplit.root"). Draw the same expression as before. Do you notice a timing difference? Is that expected?

If you still plot fE then your code might look like this:

#include "TROOT.h"
#include "TFile.h"
#include "TTree.h"
#include "TRandom.h"
#include "TClonesArray.h"
#include "myparticle.h"
#include "TH1F.h"

void read_tree()
{
   // function reading particles collection in a tree

   // open the Root file
   TFile *f = TFile::Open("/var/www/html/ROOT/exercise2_nonsplit.root");
   // get the tree from the file
   TTree *myTree = (TTree*)f->Get("MyTree");
   
   // initialize Root's timing mechanism
   gROOT->Time();
   
   // Magic of TTree::Draw()!!!!
   myTree->Draw("muons.fE", "");

   // display the time elapsed since the first call to gROOT->Time()
   gROOT->Time();
}

You should see that this runs slower. It's only due to the fact that the tree in this file is not split. So instead of reading only the part that you care about (muons.fE) ROOT has to read the full muons branch.
