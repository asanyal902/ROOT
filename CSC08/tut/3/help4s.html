2 PROOF

That worked, and it was faster - but it wasn't impressively fast. What about running the same selector on a whole cluster? We have set up a PROOF cluster for you: it uses one core of all the exercise machines. Depending on when you get here the PROOF cluster might be pretty busy or completely empty, and you will get a performance improvement that depends on the available resources.

So let's run it! The PROOF master is called "server". Any speed difference?

To start PROOF run TProof::Open("server"). It will tell you the number of nodes that it connected for you. Then inform the TChain that it should use PROOF, e.g. by adding the line ch->SetProof() to your chain setup script. Now you run the selector like before: ch->Process(...) and it will use PROOF!
