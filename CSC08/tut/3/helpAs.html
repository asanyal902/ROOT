Running a TSelector with PROOF [OPTIONAL!]

Can you find out why PROOF spawns the amount of workers it does (it should be two)?

When starting Proof on the localhost, a session with a number of workers equal to the number of CPUs in your machine will be started. As you have dual-core machines, it starts two workers:

root [0] TProof::Open("localhost")
Starting master: opening connection ...
Starting master: OK
Opening connections to workers: OK (2 workers)
Setting up worker servers: OK (2 workers)
PROOF set to parallel mode (2 workers)
