2 Reading a Tree

To accelerate the I/O for the non-split tree, we now enable TTree's pre-fetch mechanism, confusingly called 'cache'. This is disabled by default; use the documentation to find out how to enable it. Use TTree::Draw() of the same expression again (on the non-split tree). Do you notice any timing differences?

Here is the documentation for TTree. But it won't help you very much - it does not explicitly state how to enable the prefetching of data. (This will be fixed!) The trick is to simply call TTree::SetCacheSize() once! Put that line into your macro, of course after you have retrieved the TTree* and before you have read the first entry. Then simply run the macro again.
