ROOT Exercise II

Reading a TTree (2/5)

Note that for these exercises, timing is a key ingredient, so please remember the printed times. And to be sure to obtain good timing values, you have to run the macro consecutively several (at least two) times.

You can speed up the processing by enabling only the branches that you really need to read. Well, give it a try!

See the lecture on selecting which branches to access. Then see which branches you access, and tell that to the tree! The trick is to first disable everything, and then to enable only what you need. Simple, he?

Because there are sub-branches in the major branches, you cannot just set the BranchStatus for "muons" - you will have to set it for all the branches under "muons", too. You can use wildcarding for that.

Your the change in the code that selects the branch to read:

...
void read_tree()
{
   // function reading particles collection in a tree

   // open the Root file
   TFile *f = TFile::Open("http://server/ROOT/exercise2.root");
   // get the tree from the file
   TTree *myTree = (TTree*)f->Get("MyTree");
   // disable all branches 
   myTree->SetBranchStatus("*", 0);
   // enable only the branch we are interested to read
   myTree->SetBranchStatus("muons*", 1);
