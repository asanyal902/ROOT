  2. PROOF

    1. Instead of using only one core of your computer, let's use them all by running the selector in proof lite.

      SOLUTION

    2. That worked, and it was faster - but it wasn't impressively fast. What about running the same selector on a whole cluster? We have set up a PROOF cluster for you: it uses one core of all the exercise machines. Depending on when you get here the PROOF cluster might be pretty busy or completely empty, and you will get a performance improvement that depends on the available resources.

      So let's run it! The PROOF master is called "server". Any speed difference?
